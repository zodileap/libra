# Backend Contracts

这个目录保存 Libra 开源后端的结构化契约文档。

目标只有两个：

- 让其他语言的实现者可以不读 Go 源码，直接按契约重写后端
- 让仓库内后端改动有一份必须同步维护的规范基线

## 文件说明

- `backend.yaml`
  - 统一后端服务入口、目录结构、路由归口、共享环境变量、统一响应包。
- `account.yaml`
  - 账号领域契约。
- `runtime.yaml`
  - 会话与运行时领域契约。
- `setup.yaml`
  - 初始化领域契约，包括初始化状态文件和 PostgreSQL 元数据表。

## 文档结构约定

每份领域契约都应尽量保持以下结构：

- `source`
  - 指向实现来源文件。
- `storage`
  - 定义本地文件存储或数据库表结构。
- `enums`
  - 定义明确存在的枚举值与语义。
- `error_catalog`
  - 定义业务错误码、HTTP 状态码和默认外部文案。
- `schemas`
  - 定义字段级 schema，明确必填、可选、类型、默认值和约束。
- `endpoints`
  - 定义路由、鉴权方式、请求 schema、响应 schema 和可能错误。

## 同步要求

以下任一内容发生变化时，必须在同一轮改动中同步更新这里的契约文档：

- 新增、删除或修改 HTTP 路由
- 修改请求参数、响应字段或错误语义
- 修改本地存储结构、数据库表结构或默认数据文件路径
- 修改环境变量、启动命令、默认端口或部署方式
- 调整领域边界、目录结构或服务归口方式

如果代码与契约文档不一致，以代码为准，但该改动视为未完成。
