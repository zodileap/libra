# Page Implementation Checklist

## 页面结构

- 页面标题是否挂到全局 header slot。
- 是否复用了现有页面骨架、列表卡片、详情弹窗或表单组件。
- 是否为空态、加载态、错误态和禁用态预留了可见反馈。

## 交互与数据

- 按钮、菜单、弹窗和表单交互是否闭环。
- 服务层数据是否先做了归一化，再进入页面。
- 错误提示是否使用用户可读文案，而不是原始异常文本。

## 文案与主题

- 用户可见文本是否都接入 i18n。
- 样式是否只使用真实存在的主题变量。
- Light / Dark 下是否都可读、可点、可区分状态。

## 测试

- 是否补齐与本次变更最相关的单元 / UI / E2E 测试。
- 是否覆盖正常路径、边界情况和异常情况。
- 如果无法补齐测试，是否明确说明阻塞原因和风险。
