# Desktop Test Matrix

## 单元 / 结构测试

- 适合验证纯函数、资源结构、命令常量、静态回归断言。
- 速度快，优先覆盖规则和结构性约束。

## UI 测试（Playwright）

- 路径：`apps/desktop/tests/ui/playwright`
- 适合验证前端页面渲染、交互行为、文案和视觉回归。
- 不替代真实 Tauri 容器能力。

## Desktop E2E（tauri-driver）

- 路径：`apps/desktop/tests/e2e/tauri-driver`
- 适合验证 Tauri 容器、原生桥接、窗口行为和跨层集成流程。
- 成本更高，按需运行，不默认全量。

## 结果归类

- **通过**：命令真实执行且断言通过。
- **失败**：命令执行到测试断言，但存在失败用例。
- **阻塞**：测试前置条件不足，如环境、依赖、端口、二进制、权限缺失。
- **未执行**：当前轮次未覆盖，必须在结论中明确注明风险。
