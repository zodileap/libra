# OpenAPI Model Checklist

## 建模前

- 是否确认了业务实体、关键字段和调用方需求。
- 是否明确哪些字段是必填、只读、可选或枚举。
- 是否确认了错误响应和异常场景。

## 写入 OpenAPI 前

- 当前项目内是否已有 OpenAPI 文件或既定契约目录。
- 若没有，是否已确定合理的目标路径，例如 `<workspace>/docs/openapi/<project-or-module>.openapi.json`。
- 本次是创建该文件还是更新已有文件。
- 是否明确了最小接口闭环：实体、请求、响应、错误结构和 Mock 场景。

## 文件命名与路径

- 文件名是否与当前项目或模块语义一致，而不是沿用别的业务名称。
- 若项目已有其他 OpenAPI 文件，当前文件是否和它们的职责边界清晰。

## 校验结果

- 已执行：记录 OpenAPI 文件路径、命名理由和后续使用方式。
- 未执行：说明为什么没有执行。
- 被阻塞：说明阻塞点、人工替代步骤和建议。
