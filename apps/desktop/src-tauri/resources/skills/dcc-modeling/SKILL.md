# 建模执行

## 何时使用
- 用户希望执行 Blender、Maya、C4D、Houdini 等建模软件中的场景检查、对象编辑、材质处理、导入导出等操作。
- 需要把同一话题中的建模指令统一路由到某个软件，或在明确说明跨软件时规划多软件流程。

## 执行要求
1. 只能通过已注册并启用的 DCC MCP 执行软件操作，禁止伪造软件能力或跳过 MCP。
2. 如果用户明确指定软件，则优先使用该软件，并将其视为当前话题的默认软件。
3. 如果用户未指定软件且当前存在多个可用 DCC 软件，必须先要求用户选择一个软件；选择完成后，当前话题的后续操作默认继续使用该软件，直到用户明确改用其他软件。
4. 如果用户未指定软件且当前只存在一个可用 DCC 软件，可直接使用该软件执行。
5. 只有当用户在当前对话中明确提到两个或以上建模软件时，才允许直接规划跨软件流程。
6. 如果用户表达了“跨软件导出/导入/迁移”意图，但没有明确提到两个或以上建模软件，必须先让用户选择源软件和目标软件；禁止自动补全第二个软件。
7. 需要跨软件时，先生成“源软件 -> 中间格式 -> 目标软件”的迁移计划，再执行相关 MCP 调用。
8. 在执行前先确认目标对象、文件路径、导入导出格式和风险边界；缺少关键参数时先补齐信息或要求用户确认。

## 输出格式
- 当前绑定软件或跨软件计划
- 本轮拟执行的 MCP 操作步骤
- 实际执行结果与后续建议
