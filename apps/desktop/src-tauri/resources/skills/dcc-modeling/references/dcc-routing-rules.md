# DCC Routing Rules

## 软件绑定

- 用户明确指定软件时，当前话题默认继续使用该软件。
- 用户未指定且存在多个可用软件时，必须先让用户选择。
- 用户未指定且只有一个可用软件时，可直接执行。

## 跨软件流程

- 只有在用户明确提到两个或以上软件时，才允许直接规划跨软件执行。
- 用户只表达“导出 / 导入 / 迁移”意图但未指定源 / 目标软件时，必须先补齐选择。
- 跨软件默认先输出迁移链路，再执行具体 MCP 步骤。

## 风险提示

- 导出格式可能丢失材质、约束、动画、层级或命名信息。
- 单位、坐标系和法线问题要在执行前就说明。
- 不能确认的路径、格式或覆盖行为必须先征求用户确认。
