# Playwright Session Checklist

## Session Reuse

- 先确认本轮目标页面、入口地址和启动方式。
- 优先复用已有浏览器或 Electron 会话，避免每步都重启。
- 只有当主进程、启动参数或页面句柄失效时，才重建会话。

## Functional QA

- 先列出关键控件、状态切换和异常路径。
- 对每条用户可见结论都要有至少一次真实操作或断言支撑。
- 失败时优先记录第一条直接阻塞结论的异常，不要堆无关日志。

## Visual QA

- 分开检查默认态、激活态、空态、错误态和滚动状态。
- 对最终结论依赖的关键界面保留截图或页面快照。
- 明确哪些状态已经看过，哪些状态尚未验证。

## Delivery

- 报告要区分已验证、未验证和环境阻塞。
- 不要把“理论可行”写成“已确认通过”。
